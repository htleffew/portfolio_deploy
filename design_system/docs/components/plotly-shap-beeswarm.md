## What It Does
Approximates a SHAP beeswarm summary plot using Plotly scatter traces — showing global feature importance with individual prediction-level detail, color-coded by impact direction.

## ⚠️ Important: This Is an Approximation

Real SHAP beeswarm plots are generated by Python's `shap.summary_plot()` using matplotlib. This is a **browser-side JavaScript approximation** that simulates the visual pattern using random data points drawn from a normal distribution scaled by feature importance rank.

**Do not represent this as exact SHAP output.** It communicates the *structure* of SHAP interpretation (feature ranking, spread, direction) without requiring Python runtime in the browser.

When to use approximation vs. real:
- **Approximation (this component):** Portfolio articles, general audiences, when the visual metaphor matters more than exact values
- **Real SHAP:** Technical appendices, peer-reviewed contexts, when specific SHAP values must be accurate

## When to Use
- Explaining which features drive a model's predictions (interpretability section)
- Showing that a model has learned sensible feature priorities (validation)
- After presenting model performance metrics — this answers "but WHY does it perform well?"
- Any article involving XGBoost, Random Forest, or gradient boosted models

## When NOT to Use
- Linear models (use coefficient bar chart instead — simpler and more accurate)
- When fewer than 3 features matter (a bar chart is cleaner)
- When exact SHAP values are required (use Python matplotlib export as static image)

## Plotly Configuration

```js
const feats = [
  '30d_page_views',
  'demo_requests',
  'industry_SaaS',
  'revenue_tier',
  'historical_win'
];

let shapData = [];
for (let i = 0; i < feats.length; i++) {
  let xv = [], cv = [];
  for (let j = 0; j < 200; j++) {
    // Points scaled by feature rank — top features have wider spread
    let s = (Math.random() - 0.5) * 2 * (feats.length - i);
    xv.push(s);
    cv.push(Math.abs(s));  // Color by absolute impact magnitude
  }
  shapData.push({
    x: xv,
    y: Array(200).fill(feats[i]),  // All points on same y-level (feature name)
    mode: 'markers',
    type: 'scatter',
    marker: {
      color: cv,
      colorscale: 'Bluered',   // Blue (negative) → Red (positive impact)
      size: 6,
      opacity: 0.6
    },
    name: feats[i],
    showlegend: false
  });
}

Plotly.newPlot('plot-shap', shapData.reverse(), {
  paper_bgcolor: 'rgba(0,0,0,0)',
  plot_bgcolor: 'rgba(0,0,0,0)',
  font: { color: '#A1A1A6', family: "'JetBrains Mono', monospace" },
  title: { text: 'SHAP Global Feature Importance', font: { color: '#FFFFFF' } },
  xaxis: { title: 'SHAP Value (Impact on Model Output)' },
  yaxis: { title: '' },
  margin: { t: 50, b: 40, l: 150, r: 20 }  // l:150 REQUIRED for feature name labels
}, { displayModeBar: false });
```

## Critical: Left Margin

```js
margin: { t: 50, b: 40, l: 150, r: 20 }
```

The `l: 150` is mandatory. Feature names like `'30d_page_views'` are ~120px wide in JetBrains Mono at 11px. Without 150px left margin, labels are clipped. If your feature names are longer (e.g., `'sentence_repetition_rate_normalized'`), increase to `l: 200`.

## The `.reverse()` Call

```js
shapData.reverse()
```
SHAP beeswarms conventionally show the most important feature at the **top**. Plotly renders traces bottom-to-top by default. `.reverse()` before passing to `newPlot` puts the first (most important) feature at the top of the chart.

## Generalization

To adapt for a different model:
1. Replace `feats` array with your feature names (ordered most → least important)
2. Adjust spread: `(feats.length - i)` creates linearly decreasing spread. For real rank-based spread, multiply by actual importance scores.
3. Feature count: works for 3–10 features. More than 10 → reduce marker size to 4, opacity to 0.4.
